FEATURE 1: SOCIETY VOTING SYSTEM
════════════════════════════════════════

Create server/models/Poll.js:
- question: String required
- description: String default ''
- options: [{ text: String, votes: [{ userId: ObjectId ref User }] }]
- createdBy: ObjectId ref User
- deadline: Date required
- isAnonymous: Boolean default false
- status: enum ['Active', 'Closed'] default 'Active'
- timestamps true

Create server/routes/polls.js:

POST /api/polls
- Admin/superadmin only
- Body: question, description, options (array of strings), deadline, isAnonymous
- Create poll with options mapped to { text, votes: [] }

GET /api/polls
- Auth required
- Return all polls sorted by createdAt descending
- Include vote counts per option
- Include whether current user has voted

POST /api/polls/:id/vote
- Auth required (residents only)
- Body: optionIndex
- Check if user already voted in any option — if yes return 400 'Already voted'
- Push userId to that option's votes array
- If isAnonymous true — still record userId to prevent double voting but 
  don't expose who voted in GET response

PUT /api/polls/:id/close
- Admin/superadmin only
- Set status to 'Closed'

DELETE /api/polls/:id
- Admin/superadmin only
- Delete poll

Create client/src/pages/Voting.jsx:
Props: { currentUser, onNotify, theme }
Use same CSS classes: dashboard-shell, glass-panel, section-card, 
metric-card, summary-grid, button-primary, button-secondary, 
field-input, field-label, field-block, empty-state, dashboard-topbar,
BrandLogo component

State: polls, loading, showCreateForm, form

On mount: GET /api/polls

UI:

Topbar: same as other pages with BrandLogo

Hero banner glass-panel:
- Eyebrow: "Society Democracy"
- Title: "Community Voting"
- Subtitle: "Vote on society decisions transparently"

Metric cards summary-grid:
- Active Polls: count status Active
- Closed Polls: count status Closed
- Your Votes Cast: count polls where user has voted
- Total Polls: polls.length

If currentUser.role is admin or superadmin show "Create Poll" button

Create Poll Form (glass-panel, show when showCreateForm true):
- Input: Question (required)
- Textarea: Description (optional)
- Options: start with 2 option inputs, "Add Option" button adds more (max 6)
- Date picker: Voting Deadline
- Checkbox: Anonymous voting
- Submit: POST /api/polls

Polls List:
For each poll show a card:
- Question bold + status badge (Active=green, Closed=gray)
- Description if present
- Deadline: "Voting ends: June 20, 2026" or "Voting ended"
- Anonymous badge if isAnonymous
- Options as voting buttons showing:
  - Option text
  - Vote count and percentage bar
  - Highlight if current user voted for this option
- If user hasn't voted and poll is Active: clicking option votes
- If user already voted: show results only, no voting buttons
- If poll is Closed: show results only
- Admin sees "Close Poll" button on Active polls
Results shown as simple percentage bars with colors

FEATURE 3: MAINTENANCE PAYMENT TRACKING
════════════════════════════════════════

Create server/models/Payment.js:
- residentId: ObjectId ref User required
- residentName: String required
- flatNumber: String required
- month: String required (format YYYY-MM)
- amount: Number required
- status: enum ['Paid', 'Pending', 'Overdue'] default 'Pending'
- paidOn: Date default null
- method: enum ['Cash', 'UPI', 'Bank Transfer', 'Cheque'] default 'Cash'
- note: String default ''
- timestamps true

Create server/routes/payments.js:

POST /api/payments/mark
- Admin/superadmin only
- Body: residentId, residentName, flatNumber, month, amount, status, method, note
- Check if payment record exists for same residentId + month
- If exists: update it
- If not: create new

GET /api/payments/all
- Admin/superadmin only
- Query params: month (YYYY-MM format)
- Return all payments for that month
- Include total collected, total pending count

GET /api/payments/my
- Auth required (resident)
- Return all payments for logged in user
- Sort by month descending

GET /api/payments/summary/:month
- Admin/superadmin only
- Return:
  - totalResidents: count of all residents
  - paid: count paid
  - pending: count pending
  - overdue: count overdue
  - totalCollected: sum of paid amounts
  - pendingAmount: sum of pending amounts

Create client/src/pages/Payments.jsx:
Props: { currentUser, onNotify, theme }
Use same CSS classes

State: payments, loading, selectedMonth, summary

On mount: fetch based on role

If admin/superadmin:
- GET /api/payments/all?month=current month
- GET /api/payments/summary/:month

If resident:
- GET /api/payments/my

UI for ADMIN:

Topbar with BrandLogo

Hero banner:
- Eyebrow: "Financial Management"  
- Title: "Payment Tracking"
- Subtitle: "Track monthly maintenance fee collection"

Month selector input type month

Metric cards:
- Total Collected: summary.totalCollected formatted as ₹
- Paid: summary.paid residents
- Pending: summary.pending residents
- Overdue: summary.overdue residents

Payments table/list:
- Each row: Flat number, Resident name, Amount, Status badge, 
  Method, Paid on date, Note
- Status colors: Paid=green, Pending=amber, Overdue=red
- Admin can click any row to update status inline
- Update form: status dropdown, method dropdown, note input, Save button
- Calls POST /api/payments/mark

UI for RESIDENT:

Hero banner:
- Title: "My Payment History"

Metric cards:
- Total Paid: count paid
- Pending: count pending
- Total Amount Paid: sum

Payment history list:
- Each item: Month, Amount, Status badge, Method, Paid on date
- Sort newest first
