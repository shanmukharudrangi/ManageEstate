🔴 Critical Bugs (App Breaking)

BUG 1 — Marketplace images not showing

Cause: Vite dev server doesn't proxy /uploads to Express
Fix: vite.config.js needs proxy rule for /uploads

BUG 2 — Buy button instantly removes listing

Cause: Purchase route marks item "sold" immediately
Fix: Replace with "I'm Interested" flow showing 
     seller contact details
     Seller manually marks as sold

BUG 3 — Payment edit loses residentId

Cause: After populate(), residentId becomes object
       handleUpdate doesn't extract _id safely
Fix: payments.js route + Payments.jsx handleUpdate

🟡 Medium Bugs (Feature Broken)

BUG 4 — Theme toggle not working on new pages

Cause: Announcements, Voting, ResidentComplaints,
       AdminComplaints pages not applying theme correctly
Fix: All new pages need to check onToggleTheme prop
     and apply data-theme consistently

BUG 5 — Admin cannot access Marketplace

Cause: App.jsx restricts /marketplace to residents only
Fix: Allow admin and superadmin to view marketplace too

BUG 6 — Complaint resolved but resident not updated

Cause: No notification system yet
       Resident has to manually refresh to see update
Fix: After admin updates status, resident complaint 
     list should auto-reflect — move resolved complaints
     to separate "Resolved History" section

BUG 7 — Poll past deadline allowed on creation

Cause: No frontend validation on deadline date
Fix: Voting.jsx form should reject deadline 
     earlier than today

🟢 Minor Bugs (Small Issues)

BUG 8 — JWT missing name and apartment

Cause: JWT payload only has userId, role, status
       No name or apartment in token
Fix: authController.js createToken() should include
     name and apartment in payload

BUG 9 — Logout in new pages doesn't use handleLogout

Cause: Payments, Voting, Announcements pages do
       manual localStorage.removeItem instead of 
       calling proper handleLogout from App.jsx
Fix: Pass onLogout prop to all pages and use it

BUG 10 — Marketplace accessible without login

Cause: getListings route is public — no auth required
       Anyone with the URL can see listings
Fix: Add authMiddleware to GET /api/marketplace/listings
     Only logged-in society members should see listings

BUG 11 — No loading state on voting

Cause: When resident clicks vote button
       No disabled state while API call is running
Fix: Disable all vote buttons while submitting
     Show loading indicator

BUG 12 — Announcement expiry not shown clearly

Cause: Expired announcements filtered on backend
       But no visual indicator before expiry
Fix: Show "Expires in X days" warning when 
     expiry is within 2 days
