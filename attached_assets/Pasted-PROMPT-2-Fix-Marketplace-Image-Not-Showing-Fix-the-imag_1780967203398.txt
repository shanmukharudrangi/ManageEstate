Error shown: "residentId, month, and amount are required"

Resident Name, Flat Number, Amount, Status, Method, Note

which is a MongoDB ObjectId — not just the resident name.

The form currently sends residentName and flatNumber 
but NOT residentId — causing the backend validation to fail.

Fix this properly:

1. In the Payments.jsx admin form add a resident selector:
   - Fetch all residents on page load: 
     GET /api/users/residents (create this route if it doesn't exist)

     display "Name - FlatNumber" for each

     residentId (MongoDB _id)
     residentName (name)
     flatNumber (apartment)
   - Remove the manual Resident Name and Flat Number text inputs
   - Replace with this single dropdown

2. If GET /api/users/residents route doesn't exist, create it:
   - Auth required, admin/superadmin only
   - Return all users where role is 'resident' and status is 'active'
   - Return fields: _id, name, apartment

3. Make sure the POST /api/payments/mark request body includes:
   residentId, residentName, flatNumber, month, amount, status, method, note

4. Also add month field to the form — 
   an input type="month" so admin selects which month this payment is for
   Default to current month
